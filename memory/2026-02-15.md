## Decision: meeting-time-tracker should be Supabase-only
- Standardize meeting-time-tracker on Supabase (Auth + Postgres + scheduled sync). Stop mixing Firebase + Supabase.
- Rationale: relational model + RLS + migrations + scheduled jobs are a better fit; Firebase created repeated OAuth/setup friction.

## Implementation: Supabase-only cleanup committed
- `projects/meeting-time-tracker-web` standardized on Supabase server patterns and hardened migrations/RLS.
- Remaining blocker is provisioning a real Supabase project and replacing placeholder env vars.

## Browser automation decision: Chrome Relay vs Playwright CDP
- Chrome Relay requires a user gesture to attach a tab (by design).
- Attempted to attach Playwright to the *primary* Chrome profile via CDP; Chrome now requires a **non-default `--user-data-dir`** for remote debugging.
- Implemented a persistent CDP automation profile at `~/.openclaw/chrome-cdp-profile` and a launcher/LaunchAgent; user must sign in once in that profile for browser-based automation.
- Best path for Supabase is CLI-first once a Supabase PAT is provided (one-time).
